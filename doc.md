# Pokereign Guide

## 0. 项目定位

这是一个免费的宝可梦同人网页游戏，玩法参考“黑夜君临”式多人远征：
- 浏览器端 2D 俯视角游戏。
- 主要玩法是 1～3 人组队进入地图。
- 玩家在一局内捕捉、升级、进化、构筑队伍。
- 地图逐渐进入夜晚 / 缩圈状态。
- 最后全队挑战最终 Boss。
- 核心不是纯 PvP，而是 PvE / PvPvE。
- 玩家之间可以存在轻度竞争或资源博弈，但第一版优先做合作 PvE。

推荐技术路线：

Client: 
渲染：       Pixi.js (v8，性能比 v7 更好)
Tilemap：    @pixi/tilemap
摄像机：     pixi-viewport
粒子：       @pixi/particle-emitter
地图编辑：   Tiled (JSON 导出)
全屏效果：   自写合并 shader（圈+雾+昼夜一个 pass）
Server: Node.js / WebSocket

## 1. 核心单局循环

一局目标时长暂定 30 ~ 40 分钟。跟《黑夜君临》一致：白天1、夜晚1、白天2、夜晚2、终幕

玩家最开始会选择一只御三家出门，然后在探索过程中、捕捉野生宝可梦、和训练家对战、挑战大型巢穴或兴趣点，来到夜晚，生存圈逐渐缩小，面对强大宝可梦袭击。

第二天所有的野生宝可梦都会刷新，兴趣点里的怪物也会升级（匹配玩家第二天的进度），但是已清理的兴趣点不会再出现新怪物。

最终玩家会集体一起对抗终幕的神兽。

## 2. 地图交互原则

### 2.1 整体地图设计

玩家的移动速度应该是 2.5 tile / 秒，也就是 0.4 秒移动一格。地图总大小 200 x 200，也就是从地图一端走到另一端需要 80 秒。

地图中可能会有原作一样石头、水、树挡路的地方，需要有指定的秘传招式才能通过。

宝可梦中心用简单的设备取代，玩家在宝可梦中心可回满 PP 和体力。无法在宝可梦中心之间传送，除非玩家已拥有能学会“飞翔”的宝可梦。

### 2.2 兴趣点采用明雷为主

地图上的核心目标应该可见：

草丛野怪区
属性巢穴
训练师营地
进化石矿点
宝可梦中心遗迹
巨大巢穴
天气祭坛
最终 Boss 点
原因：
- 玩家需要做路线规划。
- 兴趣点是策略选择，不应该完全随机暗雷。
- 玩家看到目标后要判断“打不打、绕不绕、叫不叫队友”。

### 2.3 暗雷只用于少量事件

暗雷可以用于惊喜和风险：

稀有野怪突然出现
夜晚灾厄化宝可梦袭击
反派埋伏
宝箱怪
特殊地图事件
设计原则：

明雷负责策略，暗雷负责惊喜。

## 3. 多人行动思路

### 3.1 允许分头，但不要鼓励全程分散

推荐节奏：

前期：允许 1-1-1 分头刷资源
中期：鼓励 2-1 或集合打兴趣点
后期：必须集合打巨大巢穴和最终 Boss

### 3.2 软绑定机制

不要强制玩家始终抱团，但要用系统鼓励合作：

普通野怪：单人可打
小型巢穴：单人可打，双人更快
训练师营地：推荐双人
巨大巢穴：推荐三人
最终 Boss：全队参与
软绑定机制包括：

高级兴趣点需要多人效率更高
经验部分全队共享
倒地后需要队友救援
远离队伍时夜晚风险更高
最终 Boss 必须集合

## 4. 兴趣点设计

兴趣点不是普通宝箱，而是玩家路线决策的核心。
设计原则：

小怪给经验，兴趣点给构筑。
普通点给稳定收益，高危点给质变奖励。
兴趣点最好影响队伍流派、进化路线或最终 Boss 难度。

### 4.1 兴趣点类型

第一版推荐实现以下类型：

1. 普通野怪区
2. 属性巢穴
3. 训练师营地
4. 宝可梦中心遗迹
5. 进化资源点
6. 天气祭坛
7. 研究所 / 观测站
8. 反派据点
9. 巨大巢穴
10. 最终 Boss 点

### 4.2 属性巢穴

用途：

提供对应属性的技能机、携带物、进化资源、捕捉机会。
示例：

火焰巢穴：
- Boss: 鸭嘴火兽 / 喷火驼
- 奖励:
  - 火之石
  - 火系技能机三选一
  - 木炭
  - 火系宝可梦捕捉机会
### 4.3 训练师营地

用途：

提供招式机、金钱、道具、技能升级、泛用构筑奖励。
适合做双打战斗。
示例：

登山男营地：
- 敌人偏岩石 \ 地面 \ 格斗
- 奖励偏防御、岩石技能、地面技能、金币

### 4.4 宝可梦中心遗迹

用途：

回复 HP
复活倒下宝可梦
清除异常状态
整理队伍和携带物

### 4.5 研究所 / 观测站

用途：

提供信息奖励，而不一定是数值奖励。
奖励示例：

显示附近兴趣点
显示最终 Boss 属性
显示下一次夜晚圈
提升全队捕捉率
鉴定未知奖励

### 4.6 天气祭坛

用途：

服务天气 / 场地构筑。
示例：

雨云祭坛：
- 击败水系守卫
- 奖励三选一:
  - 求雨技能机
  - 神秘水滴
  - 队伍获得短时间雨天增益
 
### 4.7 反派据点

高风险高收益兴趣点。
特点：

敌人强
波次多
耗时长
奖励好
奖励示例：

高级球
稀有宝可梦
强力持有物
特性胶囊
进化资源

### 4.8 巨大巢穴

类似小团本。
特点：

推荐 3 人
封闭战斗区域
多阶段 Boss
奖励强
耗时长
失败惩罚明显
奖励示例：

团队共享 Buff
最终 Boss 削弱
高级进化资源
稀有招式
全队复活机会

## 5. 双打 / 三打兴趣点

不建议所有战斗都双打 / 三打。
推荐规则：

普通野怪：自由战斗
精英点：偶尔双打
训练师营地：适合双打
巨大巢穴：适合三打
最终 Boss：全队协作
设计原因：
- 全局双打会拖慢节奏。
- 新手信息量过大。
- 单人 / 双人开局会被卡住。
- 调平衡难度高。

### 5.1 双打点

适合做组合敌人。
示例：

九尾 + 叶伊布：
- 九尾开晴天
- 叶伊布高速输出
- 玩家需要先击败天气核心或处理输出点
### 5.2 三打点

适合做小团本机制。
示例机制：

三方向站位
同时打断
分头清核心
分摊伤害
Boss 护盾阶段
属性机关



## 6. 野怪设计

野怪分层：

1. 路边杂兵
2. 生态群落野怪
3. 精英野怪
4. 巢穴 Boss
5. 夜晚灾厄化野怪

### 6.1 路边杂兵

用途：

提供基础经验
少量掉落
地图活性
特点：

打得快
成群出现
不应该占用太久
### 6.2 生态群落野怪

根据区域提供属性生态。
示例：

草原：一般 / 飞行 / 草
湖边：水 / 虫 / 飞行
矿洞：岩石 / 地面 / 钢
森林：草 / 虫 / 毒 / 幽灵
火山：火 / 岩石 / 地面
雪原：冰 / 钢 / 恶
### 6.3 精英野怪

用途：

小型挑战，提供确定性奖励。
奖励示例：

技能机
进化石
高级树果
特性胶囊
属性强化道具
捕捉机会
### 6.4 夜晚灾厄化野怪

夜晚后怪物变强，作为压力来源。
特点：

攻击更高
行动更快
技能更危险
掉落更好
捕捉更难



## 7. 捕捉系统

不要照搬原作“压血、扔球、摇三下”的完整节奏。
推荐捕捉流程：

1. 玩家遇到可捕捉野怪
2. 将其打到低血量，或击破捕捉护盾
3. 野怪进入虚弱状态
4. 玩家使用精灵球 / 捕捉装置
5. 出现 1～2 秒读条
6. 成功则入队，失败则怪物逃跑或进入短暂免疫
### 7.1 捕捉对象分层

普通野怪：

可捕捉
成功率较高
入队等级 = 队伍等级 - 1
精英野怪：

击败后出现捕捉机会
成功率中等
可能需要高级球
Boss / 巨大巢穴：

一般不直接捕捉本体
击败后掉落幼体、蛋、核心、召唤券或捕捉机会
### 7.2 捕捉权

推荐个人队伍 + 队伍共享收益。
规则：

每个玩家有自己的宝可梦队伍
捕捉机会可以全队可见
玩家消耗自己的球尝试捕捉
如果队伍已满，可以替换，或将捕捉结果转化为资源
转化奖励：

经验
属性碎片
招式碎片
进化资源
金币
队伍捕捉率提升



## 8. 升级与进化系统

原作 1～100 级不适合本游戏。
本游戏采用局内 1～12 级。

Lv.1～10：正常成长
Lv.11～12：超额成长，奖励贪图玩家
### 8.1 推荐等级节点


Lv.1：开局形态，基础技能
Lv.2：解锁第二技能
Lv.3：小数值成长 / 被动强化
Lv.4：三段进化宝可梦第一次进化；单段第一次觉醒
Lv.5：招式机槽位 / 技能构筑节点
Lv.6：二段进化宝可梦进化
Lv.7：特性强化 / 携带物强化
Lv.8：三段进化宝可梦最终进化；单段第二次觉醒
Lv.9：终结技 / 大招 / 特殊形态机制
Lv.10：最终 Boss 前完全成型
Lv.11～12：额外成长
### 8.2 进化规则

三段进化：

Lv.1 初始形态
Lv.4 第一段进化
Lv.8 最终进化
二段进化：

Lv.1 初始形态
Lv.6 进化
单段宝可梦：

Lv.4 第一次觉醒
Lv.8 第二次觉醒
### 8.3 特殊进化

特殊进化统一简化为：

等级资格 + 特殊资源 / 地点 / 时间
示例：

伊布 Lv.6 + 火之石 → 火伊布
伊布 Lv.6 + 水之石 → 水伊布
伊布 Lv.6 + 雷之石 → 雷伊布
伊布 Lv.6 + 夜晚 + 羁绊 → 月亮伊布
鬼斯通 Lv.8 + 连接绳 → 耿鬼
三合一磁怪 Lv.8 + 发电站 → 自爆磁怪
### 8.4 进化触发

不建议完全自动进化。
推荐：

达到等级后获得进化资格
脱离战斗数秒后可以进化
在宝可梦中心 / 进化祭坛可以更快进化



## 9. 数值公式

第一版不使用原作完整 1～100 级公式。
保留种族值，使用局内倍率。
### 9.1 非 HP 属性


属性 = (15.5 + 种族值) × 等级倍率 + 5
适用于：

攻击
防御
特攻
特防
速度
### 9.2 HP


HP = (15.5 + HP种族值) × 等级倍率 + 50 × 等级倍率 + 5
等价于：

HP = (65.5 + HP种族值) × 等级倍率 + 10
### 9.3 等级倍率建议

第一版建议：

Lv.1   0.55
Lv.2   0.60
Lv.3   0.66
Lv.4   0.72
Lv.5   0.78
Lv.6   0.84
Lv.7   0.90
Lv.8   0.96
Lv.9   1.02
Lv.10  1.08
Lv.11  1.14
Lv.12  1.20
设计目的：

开局约等于原作 25～30 级感
最终成型略高于原作 50 级
### 9.4 进化数值

进化后直接换成新形态的种族值，再根据当前等级倍率重算。

小火龙 Lv.4 进化火恐龙：
使用火恐龙种族值 + Lv.4 等级倍率重新计算属性
不要主要靠进化额外乘算。进化的爽感主要来自：

外形变化
技能变化
定位变化
种族值提升
特性强化



## 10. 宝可梦池规模

第一版目标不是做九世代全图鉴。
推荐规模：

30～40 条进化链
覆盖 18 属性
实际宝可梦数量约 70～100 只
设计原则：

每个属性至少有 1～2 条代表线
不要按世代选，而是按玩法生态选
双属性宝可梦可以同时覆盖多个属性
角色分层：

主控宝可梦：技能完整
普通野怪：技能简化
精英 / Boss：机制特殊
不要每只宝可梦都做独占复杂技能。第一版应共用技能库和 AI 模板。



## 11. 物品系统

玩家拾取物品后，必须立刻决策。
核心规则：

玩家可以立即使用物品
可以给宝可梦携带物品
如果想揣包里，每个玩家只有 1 个背包格
这个设计用于制造即时决策压力，避免玩家囤积资源。
### 11.1 物品处理方式

拾取物品后，玩家必须选择：

1. 立即使用
2. 给当前宝可梦或队伍宝可梦携带
3. 放入唯一背包格
4. 分解 / 丢弃转化为少量资源
### 11.2 物品类型

立即使用型：

伤药
活力碎片
经验糖果
临时强化药
捕捉率提升道具
携带型：

木炭
神秘水滴
奇迹种子
生命宝珠
讲究围巾
贝壳之铃
吃剩的东西
消耗携带型：

文柚果
解毒果
抗火果
抗水果
抗电果
关键资源型：

技能机
进化石
连接绳
天气核心
Boss 对策道具
特殊钥匙
### 11.3 树果设计

树果可以三用：

立即使用：立刻回复 / 解除异常
携带：满足条件自动触发
背包：之后手动使用
示例：

文柚果：
- 立即使用：回复 25% HP
- 携带：HP 低于 50% 时自动回复 25%
- 背包：保留，之后给任意宝可梦使用
### 11.4 技能机设计

技能机处理方式：

1. 立刻教给宝可梦
2. 放进唯一背包格
3. 分解成招式碎片 / 经验
技能机不允许无限储存。
### 11.5 进化石设计

进化石处理方式：

1. 立刻给可进化宝可梦使用
2. 放进唯一背包格
3. 分解成属性碎片 / 属性强化
进化石可以形成路线赌博：

玩家拿到雷之石，但当前没有伊布或皮卡丘：
- 留着赌后面抓到合适宝可梦
- 现在分解换收益
- 放弃

## 13. 对战核心机制

对战需要完全按照第九代的最新数值、机制设计。技能威力、PP、异常状态等都需要和原作一致，建议直接使用 pokemon-showdown 开源的对战模拟器。
在此基础上，因为游戏有缩圈机制 + 现实时间，所以缩圈之后，宝可梦应该在每一个逻辑帧损失血量，被招式攻击后额外损失血量。
